Part I

How we found it.

01 — Where it started

We set out to study a different bug.

Every result here began as a side-quest from a project that was about to fail. The failure is what showed us where to look.

Modern AI assistants no longer start each conversation from a blank slate. After every session they write notes to themselves — a running MEMORY.md file the model authors, re-reads, and rewrites over and over. It is the agent's diary, and everything it later believes about you is drawn from it. We were studying how that diary degrades over time: each time the agent re-summarizes its own notes, does the meaning quietly drift? We called it “Soul Drift.”

It was a reasonable hypothesis, and at first the data cooperated. Small and mid-tier models did drift as they rewrote their memory — facts blurred, emphasis shifted. We built a whole measurement harness around it: synthetic event streams, a held-out judge, preregistered probes. We thought we had a paper.

Why memory at all? Because persistent memory is the fastest-growing piece of agent infrastructure in 2026 — and the one with the least scrutiny. Whatever an agent writes to memory silently shapes every future decision it makes for you.
02 — The moment it broke

The best model showed zero drift.

When we ran the same test on a frontier model, the effect didn't shrink. It disappeared.

We had been testing on open, mid-tier models. The instant we put the strongest model in the world at the time — DeepSeek V4-pro — through the identical pipeline, the drift was flat zero. Where a mid-tier model smeared its memory over many rewrites, the frontier model held its notes perfectly stable. The effect attenuated with capability: the better the model, the less it drifted.

That is fatal for a research paper. A claim like “this matters for frontier models like GPT-5” dies the moment a reviewer points out the effect vanishes exactly where it would matter. A weakness that shrinks as models improve is a weakness that's already solving itself. We had measured something real — and watched it become irrelevant.

A failure that fades as models get smarter is not a frontier story. We needed the opposite.
03 — The insight that flipped the search

Hunt for a weakness that grows with capability.

Instead of a bug that capability cures, we went looking for a bug that capability causes.

The pivot was a single inversion. The most interesting — and most publishable — failures are the ones that require intelligence: deception, scheming, over-reasoning, situational awareness. These don't appear in weak models because weak models aren't capable enough to commit them. They emerge with capability. If we could find a memory failure of that kind, it would survive at the frontier by definition — it would be a property of the frontier.

So we reframed the entire question. Not “how does memory degrade?” (an omission story that favours small models) but “what does a capable model actively put into memory that doesn't belong there?” — a commission story that should favour large ones. A smarter model reasons more, infers more, fills gaps more. What if it does that while writing its own memory, and files the result as fact?

Commission vs omission. Omission = forgetting things that were true (small models do this more). Commission = recording things that were never said (we bet large models would do this more). The whole pivot rides on that distinction.
04 — The hunt

A swarm of agents read the literature for us.

To find an unclaimed failure of exactly this shape, we ran a multi-agent search across the field.

We launched a fan-out of research agents over roughly 190 papers from ACL, NeurIPS, ICLR and recent arXiv, each hunting for capability-dependent failure modes, then put every candidate through adversarial refutation — other agents trying to prove the idea was already published. Most candidates died. One survived every refuter and matched the pivot perfectly: agents writing unsupported inferences as fact into their own persistent memory.

It had everything we wanted: it should require capability (you must be smart enough to infer before you can over-infer), it lived in our existing harness, and — critically — no one had measured it. We named it provenance collapse: the boundary between what the user stated and what the model inferred collapses inside the agent's own memory.

What it is.

06 — How the agent behaves

Watch the read → infer → write loop.

The animation below loops the agent's actual behaviour: it reads a messy human message, silently infers to fill the gap, and commits the inference to memory.

Figure 2 The behaviour, animated (loops). The agent reads an under-specified message, forms an inference in its reasoning (amber), and writes that inference into memory as an unqualified fact (coral). The same helpfulness that makes it a good chat partner makes it a bad archivist.

Why does it happen? Faced with a vague human message, a capable model does what it was trained to do — reason, infer, smooth over gaps. In a live reply that is helpful. Written into permanent memory as fact, it is corruption that compounds: every future session reads the invented claim as ground truth and builds on it.

Two precise definitions

Commission, not omission. Existing memory research measures forgetting — facts that get lost. Provenance collapse is the reverse: facts that get invented. Formally, a written claim is “collapsed” when it is not entailed by the conversation and written without any hedge, as flat fact.

The metric. We decompose each memory note into atomic claims, label every claim against the transcript (entailed / unsupported / contradicted), and report collapse rate = unsupported-&-unhedged claims ÷ total claims.

Part IV

What we found.

09 — The headline result

The smarter the model, the more it invents.

A small base model collapses on 7% of its claims. Every larger, frontier, or reasoning model — across five vendors — lands at 22–40%.

Figure 4 · Collapse rate by model

Holds across five independent vendors — Meta, OpenAI, Mistral, DeepSeek, Qwen, Zhipu. Not one lab's quirk.

Persists at the very frontier. GPT-5 (29.8%) and DeepSeek V4-pro (30.1%) are among the worst — the models people trust most. Capability does not cure it.

A threshold, stated honestly. The jump is from “small base model” to “everything capable.” Within a family the climb is directional but not strictly monotone — so we call it a capability threshold, not a smooth dial.

10 — The mechanism, visible

Give it a gap, and it fills it.

Inside every model, collapse is near-zero when the user states things plainly — and explodes when the input is vague. The fingerprint of gap-filling.

Figure 5 · Collapse rate by input ambiguity

Low-ambiguity inputs (facts stated outright) yield almost no collapse; vague, emotional inputs send capable models to 40–56%. The effect tracks how much there is to infer.

Part V

How to stop it.

12 — The fix

One paragraph of instruction nearly erases it.

A “provenance-disciplined” prompt — record only what was stated; mark inferences as inferred; omit when unsure — cuts collapse 60–97%.

Figure 6 · Baseline vs. provenance-disciplined prompt

Paired on identical sessions. Reductions significant at McNemar p ≤ 0.0034 for all three models.

“But did it just write less?”

The natural worry: maybe it cut inventions only by writing fewer notes, dropping real facts too. So we measured stated-fact recall: of what the user genuinely said, how much survives?

Figure 7 · Stated-fact recall — baseline vs disciplined

Recall stays within ~3 points (every paired-difference CI includes zero) while collapse drops 60–97%. The fix removes inventions, not facts. Confound closed.

The paper.

The working ACL submission, in its own format — abstract, sections, figures, and references. Every number traces to a run in the footer.

Inference as Fact: Capability-Scaled Provenance Collapse
in Self-Authored Agent Memory
VelaOps Research
working draft · target: ACL 2027 (main)
Abstract. LLM agents increasingly maintain their own persistent natural-language memory — notes the model writes to itself and re-reads across sessions. We identify and measure a write-time failure mode we call provenance collapse: the agent records unsupported inferences as flat declarative facts the user never stated. This is a commission error, distinct from the omission/recall failures existing memory benchmarks measure. Across 30 controlled sessions and 10 models from 5 vendors, provenance collapse scales with model capability: a small base model (Llama-4-17B) collapses on 7.1% of self-authored claims, while every larger, frontier, or reasoning model — including GPT-5 and DeepSeek-V4-pro — collapses on 22–40% (all Fisher one-sided p ≤ 0.0003). The effect is a capability threshold that persists at the frontier and grows with input ambiguity. Our metric is validated by cross-family judge agreement (Cohen's κ = 0.63–0.68). Finally, the failure is largely fixable at the prompt level: a provenance-disciplined instruction reduces collapse 60–97% (McNemar p ≤ 0.0034) while preserving stated-fact recall.

1Introduction

Persistent memory is becoming standard agent infrastructure: assistants summarise each session into durable notes that condition all future behaviour. The implicit assumption is that what the agent writes is what it was told. We show this fails systematically, and worse in more capable models. We define provenance collapse as the event in which a model writes, into its persistent memory, a claim not entailed by the session it summarises, as if it were stated fact (Figure 1). Our contributions: (i) the phenomenon and a validated claim-level metric; (ii) a capability-scaling law across 10 models and 5 vendors; (iii) a prompt-level mitigation.

2Related Work

Concurrent work (MemIR; Jin et al., 2026) names “provenance-role collapse” and proposes a typed-memory architecture, but reports no baseline rate and no capability scaling. Memory benchmarks (LoCoMo; LongMemEval; MemoryAgentBench) measure recall of stated information, not write-time fabrication. PrefEval measures adherence to stated preferences — the converse. Inverse scaling and TruthfulQA document larger-is-worse on prompted QA truthfulness, a distinct setting. Ours is the first measurement of commission-vs-capability in self-authored memory.

3Method

We construct 30 sessions at three ambiguity levels. Each model consolidates each session into memory. A held-out judge from a different model family decomposes every note into atomic claims and labels each entailed, unsupported, or contradicted against the transcript (Figure 3). Collapse rate = unsupported-and-unhedged claims ÷ total claims. Controls: ≥30 claims/cell, paired tests, prompt-paraphrase arm, cross-family judge.

4Results

Collapse rises from 7.1% (Llama-4-17B) to 22–40% across nine larger/frontier/reasoning models, every contrast Fisher p ≤ 0.0003 (Table 1, Figure 4). It persists at the frontier (GPT-5 29.8%, V4-pro 30.1%) and grows with ambiguity (Figure 5): within Qwen3-32B, 8.9% → 44.1% → 55.9% from low to high. The metric is reliable across families (κ 0.63–0.68).

5Mitigation

A provenance-disciplined consolidation prompt cuts collapse 60–97% (Qwen3-32B 36.1→1.0%, Llama-3.3-70B 24.7→4.7%, GPT-5-mini 16.5→6.5%; McNemar p ≤ 0.0034; Figure 6). Stated-fact recall is preserved within 3 points (Figure 7), ruling out a “wrote-less” confound.

6Limitations

Absolute magnitude is prompt-sensitive; the capability ordering is robust but within-family steps are not individually significant (a threshold, not a smooth law). Labels are judge-based (human validation in progress). Sessions are synthetic; naturalistic deployment is future work.

7Conclusion

Provenance collapse is, to our knowledge, the first LLM-agent memory failure shown to intensify with capability rather than fade — measurable, cross-vendor, present at the frontier, and substantially fixable with a single prompt.

14 — What we're not claiming

The honest fine print.

Magnitude is prompt-sensitive. Absolute rates shift with consolidation wording (one model fell to 2.4% under one phrasing). The capability ordering holds; we report per-prompt.
A threshold, not a smooth law. Within-family frontier steps are directional but not individually significant.
Judge-based labels. Validated by cross-family agreement (κ 0.63–0.68); gold human labels still being collected.
Controlled, synthetic sessions. 30 hand-built transcripts; naturalistic long-horizon deployments are next.
15 — Why it matters
As we hand agents durable memory and more capability, they don't just remember better — they remember things we never said, and write them down as truth.

Provenance collapse is the first LLM-agent memory failure shown to intensify with capability rather than fade. Measurable, cross-vendor, present at the frontier — and substantially fixable with a single prompt. A novel phenomenon, a validated metric, a capability law, and a working mitigation.
